1 Building Abstractions with Functions

1.1 The Elements of Programming

  • expressões primitivas: que representam as entidades mais simples da linguagem.
  • meios de combinação: pelo qual os elementos compostos são construídos a partir dos mais simples.
  • meios de abstração: pelo qual os elementos compostos podem ser nomeados e manipulados como unidades.

- As expressões que representam números podem ser combinadas com operadores (como + ou *) para formar uma expressão composta que representa a aplicação de uma função primitiva correspondente a esses números. Por exemplo:

- Como de costume, os parênteses são usados para agrupar combinações de operadores para evitar ambiguidades:

(3 * 5) + (10 - 6);

- O JavaScript também segue as convenções usuais quando os parênteses são omitidos: a multiplicação e a divisão antes de adição e a subtração. Por exemplo:

3 * 5 + 10 / 2;

- ou:

(3 * 5) + (10 / 2);

1.1.3 Evaluating Operator Combinations

  1. Avalie as expressões as combinação de operadores.
  2. Aplique a função que é denotada pelo operador aos argumentos que são os valores dos operandos.
(2 + 4 * 6) * (3 + 12);
  • os valores dos numerais são os números que eles nomeiam, e
  • os valores dos nomes são os objetos associados a esses nomes no ambiente.

1.1.4 Compound Functions

Elementos que aparecem em toda linguagem poderosa:

  • Números e operações aritméticas são dados e funções primitivas.
  • O aninhamento de combinações fornece um meio de combinar operações.
  • Declarações constantes que associam nomes a valores fornecem um meio limitado de abstração.

“To square something, take it times itself.” “Para elevar algo ao quadrado, multiplique ele por ele mesmo.”

function square(x) {
  return x * x;
}
function square(   x   ) { return x   *     x; }
   |        |      |              |   |     |
   v        v      v              v   v     v
  To      Square  Something, take it time itself.

Temos aqui uma funcão composta.

A maneira mais simples para declarar uma funcão é:

function nome(parametros) { return expressao; }
  • nome: é um simbolo associado a definicão da funcão;
  • parametros: sao os nomes usados no corpo da funcão para se referir ao seu argumento correspondente
  • corpo: o corpo dessa funcão é uma simples expressão de return

Aplicacão de funcão:

square(21);
441

Depois de combinacões de operadores, aplicacões de funcões é o segundo tipo de combinacão de expressões que encontramos. A forma geral para aplicacão de funcão é:

function-expression(argument-expressions)

Para avaliar a aplicacão de uma funcão:

  1. Avalie as subexpressões do aplicacão, ou seja, a expressão de função e as expressões de argumento.
  2. Aplique a função que é o valor da expressão de função aos valores das expressões de argumento.
square(2 + 5);
49;

square(x) + square(y);

function sum_of_squares(x, y) {
  return square(x) + square(y);
}

sum_of_squares(3, 4);
25;

function f(a) {
  return sum_of_squares(a + 1, a * 2);
}
f(5);
136;

1.1.7 Example: Square Roots by Newton’s Method

Importante diferenca entre funcões matemáticas e funcões de computadores. Funcões de computadores devem ser eficientes.

Funcão matemática pra raiz quadrada:

√x = the y such that y ≥ 0 and y 2 = x

Computador:

function sqrt(x) {
  return the y with y >= 0 && square(y) === x;
}

A diferenca está na forma de descrever as propriedades das coisas e como fazer as coisas. Essa é a diferenca entre declarativo e imperativo.

Funcões matemáticas são declarativas (o que fazer) e computadores imperativos (como fazer).

1.1.8 Functions as Black-Box Abstractions

Estratégia de decompor o problema em partes menores onde cada funcão realiza uma tarefa identificável que possa ser usada como um módulo na definicão de outras funcões.

A funcão is_good_enough é uma caixa preta, de cara nós não precisamos entender o que ela faz, apenas saber que ela computa a raiz quadrada de alguma forma.

Local names

Nomes usandos como parametros sao locais ao corpo da funcão e nao devem ser palavras reservadas ou nomes de funcoes ja existestem.

Internal declarations and block structure

Declaracões de funcões internas para evitar conflitos.

function sqrt(x) {
  function is_good_enough(guess, x) {
    return abs(square(guess) - x) < 0.001;
  }
  function improve(guess, x) {
    return average(guess, x / guess);
  }
  function sqrt_iter(guess, x) {
    return is_good_enough(guess, x) ? guess : sqrt_iter(improve(guess, x), x);
  }
  return sqrt_iter(1, x);
}

Cada par de chaves são um block.
